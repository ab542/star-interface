## 项目经验：Star Digi+ 广告平台接口自动化测试框架

### 项目概述
Star Digi+ 是一个跨境电商广告管理平台，本项目负责平台后端接口的自动化测试，保障接口功能正确性和稳定性。项目采用**数据驱动测试架构**，所有测试用例由YAML定义，通用测试运行器自动执行，支持从OpenAPI自动生成测试用例。

### 技术架构
- **测试框架**：Pytest + requests + loguru
- **用例存储**：YAML文件按模块目录组织，目录名自动映射为pytest marker
- **报告输出**：pytest-html + Allure，解决Windows下中文乱码问题
- **重试机制**：针对5xx服务端错误自动重试，单个用例可配置重试次数
- **日志分离**：成功/失败请求分开记录，便于问题定位

### 核心贡献

1. **框架设计与实现**
   - 设计数据驱动测试架构，测试用例与执行逻辑分离，用例维护成本大幅降低
   - 实现**五层断言防御机制**：网络异常捕获 → 数据库关键字检测 → 系统异常检测 → 业务字段精确匹配 → 业务消息异常检测，层层把关不放过任何缺陷
   - 针对预期错误场景做智能优化，当预期业务码非0时自动跳过消息检查，适配项目错误处理模式

2. **测试用例设计方法论**
   - 提出"**每个筛选参数独立测试**"的设计原则，每个筛选条件单独编写测试用例，确保能独立工作
   - 完整覆盖：基础场景 + 单参数测试 + 边界测试 + 特殊值测试，保证测试覆盖率
   - 处理接口依赖场景：总结企业级接口依赖的五种处理方案，本项目采用测试数据预准备方案，简单可靠

3. **工程化建设**
   - 创建OpenAPI文件目录规范，统一存放接口文档，便于从OpenAPI自动生成测试用例
   - 配置pytest自动marker，按照目录结构自动给用例打标记，支持按模块选择性执行
   - 解决pytest-html报告在Windows下中文乱码问题，输出UTF-8 with BOM格式保证中文正常显示

4. **测试数据管理**
- 从OpenAPI自动生成结构化测试用例，顶部添加分类统计注释，清晰展示用例分布
- 接口依赖处理：先调用依赖接口获取真实ID，更新到测试用例中，保证测试数据有效性
- 支持变量占位符替换，`$projectId`自动从配置文件读取，敏感信息不进入代码仓库

### 项目成果
- 已实现 `login` / `monitor` / `monitor_center` / `workbench` 多个模块，共计 100+ 接口测试用例
- 所有用例**全部执行通过**，能够每日回归，有效拦截接口缺陷
- 新增接口测试用例效率提升：只需要编写YAML，无需修改Python代码，平均一个接口只需要5-10分钟
- 框架可扩展：支持任何新增模块，只需要创建目录即可，符合开闭原则
